# dictor 字典总览

整理后的字典库按 **实战精选 + 三大目录扩展库** 组织。
**新用户先看 [`_实战精选/`](_实战精选/README.md),38 个能打的够 90% 场景。**
精选不够用时按 [`扩展库索引.md`](扩展库索引.md) 找具体文件。

## 顶层结构

| 目录 | 职责 | 内容 |
|---|---|---|
| `_实战精选/` | **即开即用** | 38 个精选字典(密码/目录/用户名/Payload),~17 万行 |
| `passwords/` | **凭据(扩展)** | 密码、用户名、服务口令、默认口令、通用爆破词表 |
| `directories/` | **路径(扩展)** | 目录扫描、技术栈路径、产品路径、文件类型路径 |
| `web-specific/` | **Web 测试专项(扩展)** | 攻击载荷、参数/扩展名/文件枚举、鉴权绕过、模糊测试 |

**扩展库统计**:524 个文件 / 1132 万行 / 145MB(详细使用场景见 [`扩展库索引.md`](扩展库索引.md))

## passwords/ 子目录

- `常用密码/` — 通用弱口令、字典合集（45 文件）
- `用户名/` — 用户名、姓名、邮箱、手机号（18 文件）
- `服务口令/` — 各服务（SSH/RDP/MySQL/Redis 等）默认用户名+口令对（90 文件）
- `默认口令-设备/` — 路由器、安全设备等硬件默认口令
- `特殊密钥/` — JWT、webshell 密码

## directories/ 子目录

- `通用/` — 通用目录字典、robots 禁用路径、Linux/Win 系统路径
- `后台/` — 后台路径字典
- `备份/` — 备份文件字典
- `技术栈-ASP/` `技术栈-ASPX/` `技术栈-JSP/` `技术栈-PHP/` `技术栈-文件类型/` — 按技术栈分类
- `产品-Spring应用/` — Spring Boot Actuator/Swagger/Druid 端点
- `产品-平台组件/` — Apache/Tomcat/WebLogic/IIS/Oracle/SAP 等中间件路径
- `产品-网站系统/` — WordPress/phpMyAdmin/editor 等产品路径
- `产品-网站系统/CMS漏洞路径/` — 各 CMS 0day/路径漏洞（discuz/ecshop/wordpress 等）

## web-specific/ 子目录

- `攻击载荷-SQL注入/` `攻击载荷-XSS测试/` `攻击载荷-上传测试/` `攻击载荷-文件包含与执行/` `攻击载荷-模糊测试/` — 各类 Payload
- `枚举-参数/` `枚举-扩展名/` `枚举-文件/` — 参数/扩展名/文件名枚举
- `鉴权绕过/` — User-Agent、Header、HTTP 方法等绕过字典
- `杂项/` — debug/test/error 等通用路径杂项
- `资源/` — 非字典资源（xlsx 速查表、rar 密码包、poc 样本）
- `说明/` — 字典来源许可证（fuzzdb、SVN-Digger 等）

## 使用顺序

1. **打开 `_实战精选/`** → 90% 场景直接能用,不用纠结选哪个
2. **需要更广覆盖** → 进 `passwords/常用密码/` 取 100 万行级别大字典
3. **专项爆破** → `passwords/服务口令/` 取 SSH/RDP 等服务的 -user/-pass 配对
4. **特定技术栈** → `directories/技术栈-*/` 或 `产品-*/`
5. **深度 Payload** → `web-specific/攻击载荷-*/`

## 整理原则

- 每个目录**职责单一**，无跨目录重复
- 所有 .txt 文件已**行级去重**（trim 行尾空白、忽略大小写去重、按字母排序）
- 同名文件冲突时合并内容并去重，不再保留多份
